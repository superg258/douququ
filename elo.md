我建议你从头改成一套“阶段感知的正则化 Elo”，而不是继续在单层 Elo 上调 K。因为你现在遇到的不是普通噪声，而是赛制带来的结构性偏差：RMUC 本身就是区域赛→复活赛→全国赛的分层晋级结构，区域赛优胜队还能直接进全国赛；而官方积分体系也明确写了，同一高校在同一赛季如果在 RMUC 多个阶段都有成绩，只取更高的积分值，而不是让更高阶段的失利把它再拉回去。这个逻辑本质上就是在防“晋级反而吃亏”，你现在的评分系统缺的正是这层保护。([RoboMaster][1])

你的权重方向本身是合理的。官方积分体系本来就强调近三季衰减，且分数是绑定高校而不是严格绑定某一届具体阵容；同时官方把 RMUC 视为顶级赛事，而 RMUL 的规则难度、机器人数量、人力和资金需求都更低，更像进阶前的积累场。因此，2025RMUC 应该是主锚，2026RMUL+三月底形态是近期状态补充，2024RMUC 是长一点的回看，历史排名只做弱先验。这个大框架没有问题，问题是这些信息不能被你全都当成“对局结果”去顺序更新。([RoboMaster][2])

技术上，我建议保留 Elo 的胜率骨架，但加两个 Glicko / Bayesian 风格的部件。第一个是不确定性 (\sigma)：Glicko 明确把 rating 的可靠性单独建模，比赛会降低不确定性，长期不打会提高不确定性。第二个是批量校准初值：TrueSkill Through Time 指出，只做单向 forward 更新会让初始 skill estimate 不稳定；TrueSkill2 也说明，把 winner/loser 之外的额外信息并入 skill 推断，往往能显著提升预测质量。这正适合你手里的“综合表现”和“完整形态”。

下面我直接给你一套完整方案。

一，先定义你真正要维护的量。

每支队伍维护三个值：

[
\mu_i
]

表示当前平均实力，也就是你最后展示给用户看的 Elo 分。

[
\sigma_i
]

表示这支队伍这个 Elo 有多可靠。数据越新、比赛越多，(\sigma) 越小；长期没打或只有历史排名，(\sigma) 越大。

[
F_i
]

表示“阶段认证下界”。它不是 Elo 本体，而是这支队伍在本赛季已经证明过自己的最低档位。这个量专门解决“国赛 0-3 被打成弱队”的问题。

二，赛前先验，不要用假比赛，直接做加权锚点。

把你所有来源先统一成同一尺度。最稳妥的是都转成 percentile，再转成 ([-1,1])：

[
q_{i,s}=1-\frac{\mathrm{rank}*{i,s}-1}{N_s-1}, \qquad z*{i,s}=2q_{i,s}-1
]

如果某个来源不是名次，而是综合表现分，就先在该来源内部做 percentile，再转成 (z)。

你给的总权重，我建议拆成下面这 6 项：

[
0.35 \cdot z_{25game}
+0.10 \cdot \tilde z_{25perf}
+0.25 \cdot z_{26rmul}
+0.15 \cdot \tilde z_{form}
+0.10 \cdot z_{24}
+0.05 \cdot \tilde z_{hist}
]

这里我故意把 2025RMUC 和 2026RMUL+形态拆开了，因为“逐局赛果”和“综合表现”高度相关，“RMUL赛果”和“完整形态”也高度相关。后面三个带波浪号的项，不建议直接用原始值，而建议做“残差化”之后再用：

[
\tilde z_{25perf}=\mathrm{resid}(z_{25perf}\sim z_{25game})
]

[
\tilde z_{form}=\mathrm{resid}(z_{form}\sim z_{26rmul})
]

[
\tilde z_{hist}=\mathrm{resid}(z_{hist}\sim z_{24},z_{25game},z_{26rmul})
]

意思很简单：综合表现里那些已经被逐局战绩解释掉的部分，不要再算第二遍；完整形态里那些已经被 RMUL 赛果解释掉的部分，也不要再算第二遍；历史排名里那些只是“老牌强校惯性”的部分，也别重复放大。

赛前总锚点：

[
H_i=
0.35 z_{25game}
+0.10 \tilde z_{25perf}
+0.25 z_{26rmul}
+0.15 \tilde z_{form}
+0.10 z_{24}
+0.05 \tilde z_{hist}
]

转成初始 Elo：

[
\mu_i^{(0)} = 1500 + 240 H_i
]

`240` 这个 spread 先别设太大。你之前出现“预测很怪”，大概率就是赛前分布太陡、模型太自信。

初始不确定性可以这样给：

[
\sigma_i^{(0)}=
\mathrm{clip}!\left(
100/\sqrt{1+n_i^{eff}} + 15\cdot \mathbf 1[\text{缺少2026RMUL}],
45,110
\right)
]

这里 (n_i^{eff}) 是有效样本量。2025RMUC 逐局和 2026RMUL 这种直接对局数据权重大；2024RMUC 次之；历史排名最小。你也可以更简单一点：有 2026RMUL + 2025 逐局的队给 45–60；只有 2025 的给 65–80；只剩老数据/历史榜的给 90–110。

三，赛内更新，Elo 只吃“真实对局”。

赛内不要让“综合表现”“完整形态”“历史排名”再参与逐场更新。它们只属于先验层和锚点层。

对局胜率仍然用 Elo 的 logistic：

[
E_{ij}=
\frac{1}{1+10^{-\Delta_{ij}/400}},
\qquad
\Delta_{ij}=\mathrm{clip}(\mu_i-\mu_j,-300,300)
]

我建议把 rating difference 截断在 (\pm 300)。这样最大赛前胜率大约 85%，能显著减少模拟时的离谱碾压。

更新公式：

[
\mu_i \leftarrow \mu_i + K_s (S_i-E_{ij})
]

其中 (S_i\in{1,0})。

推荐的阶段 K：

区域赛 (K=18)
复活赛 (K=12)
全国赛小组赛 (K=10)
全国赛淘汰赛 (K=14)

逻辑是：区域赛信息量大，K 稍高；复活赛/全国赛尤其是小组赛，样本少且强队密集，K 要保守。淘汰赛比小组赛更能区分顶层队，所以略高一点。

如果你手里是分局数据，而一场 series 有多局，不要把每局当完全独立样本。建议每局只用：

[
K_{\text{map}} = K_s/\sqrt{M}
]

其中 (M) 是 series 总局数。这样 BO3 的总信息量大约是 BO1 的 1.73 倍，不是 3 倍。

(\sigma) 的更新可以做得很轻：

每打完一个 series，

[
\sigma_i \leftarrow \max(35, 0.95 \sigma_i)
]

赛段间隔较长或跨赛季时，

[
\sigma_i \leftarrow \min(110,\sqrt{\sigma_i^2 + c^2\Delta t})
]

这一步是借 Glicko 的思想，不需要你完整上 Glicko 公式，也足够实用。

四，最关键的部分：赛段保护机制。

这是整套方案的核心。没有它，国赛 0-3 会继续把队伍打废。

思路是：晋级本身就是正向证据，而且是非常强的正向证据。你不能只让“后续三场失利”参与更新，而让“它先打进了更高赛段”完全不入模。

我建议这样做。

先在每个赛段开始前，记录该队进入该赛段时的分数：

[
\mu_i^{in,s}
]

然后按正常 Elo 把这个赛段的比赛打完，得到“纯比赛更新后的分数”：

[
\mu_i^{match,s}
]

接着，根据该队在这一赛段获得的“认证等级”，给它一个阶段下界 (F_i^s)。

这个下界不要用全局固定加分，而要用“本赛区/本赛段的切线分数”来算。也就是按赛段结果，找晋级边界两侧那两支队在赛段开始前的 rating，取中点。

例如某个区域赛有直通全国赛名额 (M)，那么：

[
C_{dir,r}=
\frac{
\mu_{\text{最后一个直通队}}^{in,reg}
+
\mu_{\text{第一个未直通队}}^{in,reg}
}{2}
]

同理，复活赛资格线也算一个：

[
C_{rep,r}=
\frac{
\mu_{\text{最后一个复活赛资格队}}^{in,reg}
+
\mu_{\text{第一个淘汰队}}^{in,reg}
}{2}
]

然后给不同认证一个软下界：

直通全国赛：

[
F_i^s=C_{dir,r}+15
]

进入复活赛：

[
F_i^s=C_{rep,r}+8
]

复活赛突围到全国赛：

[
F_i^s=C_{nat,\mathrm{rep}}+10
]

这里不是固定“晋级 +50”，而是“围绕本地切线给一个认证下界”。这样会自动适应不同赛区强度，不会把弱赛区直通队抬得过高，也不会把强赛区边缘队压得过低。

最后做阶段 shrinkage，而不是直接采用 (\mu_i^{match,s})。

[
\lambda_i^s = \frac{n_i^s}{n_i^s+n_0^s}
]

[
B_i^s = \max(\mu_i^{in,s}, F_i^s)
]

[
\mu_i^{out,s} = \lambda_i^s \mu_i^{match,s} + (1-\lambda_i^s) B_i^s
]

其中 (n_i^s) 是该赛段有效 series 数。推荐：

区域赛 (n_0=4)
复活赛 (n_0=5)
全国赛小组赛 (n_0=6)
全国赛淘汰赛 (n_0=4)

这是什么意思？
就是样本少的赛段，不允许它把前面的证据全部覆盖掉。

举个你最关心的例子。某队进入全国赛时 (\mu^{in}=1600)，全国赛小组 0-3，按纯 Elo 打完之后 (\mu^{match}=1568)。因为小组赛只有 3 场，取 (n_0=6)，则：

[
\lambda = 3/(3+6)=0.333
]

如果它的阶段认证下界 (F=1585)，那么

[
B=\max(1600,1585)=1600
]

最后：

[
\mu^{out}=0.333\times 1568 + 0.667\times 1600 \approx 1589
]

也就是它会掉分，但不会因为 3 场国赛失利就被打成一支明显弱队。这个行为正是你想要的。

五，这套机制为什么比“给国赛输球设更小 K”更好。

因为单纯调小 K，只是在减慢下跌速度，但没有把“晋级”这条正向信息写进模型。它只是个补丁，不是机制。

而我上面这套写法，等价于给模型加了一条规则：

“更高赛段的失利，只能在已经确认它打进更高赛段这个事实的前提下，修正它有多强；不能反过来否定它打进更高赛段这件事本身。”

这才是真正消除“晋级吃亏”。

六，预测与模拟时，不要只用点估计。

你前面说“模拟赛果时不可靠”，这通常不只是 Elo 均值的问题，还是你把均值当真值的问题。

模拟时，每支队先采样：

[
r_i \sim \mathcal N(\mu_i,\sigma_i^2)
]

然后用采样后的 (r_i,r_j) 算胜率：

[
p_{ij}=\frac{1}{1+10^{-(r_i-r_j)/400}}
]

这样做有两个好处。

第一，资料充分、状态清晰的队，(\sigma) 小，模拟更稳定。
第二，只有历史排名支撑、近期资料很少的队，(\sigma) 大，模型不会过分自信。

你最终展示给人的 Elo 分数仍然用 (\mu_i)；但实际做 Monte Carlo 模拟时，一定要用 ((\mu_i,\sigma_i)) 而不是只用 (\mu_i)。

七，回测别再只看命中率，要加两个你这个任务特有的指标。

第一，看时间顺序上的 log loss。
你要做的是概率模型，不是单纯猜对猜错。

第二，专门加一个“晋级吃亏率”：

[
\text{晋级吃亏率}
============

\frac{
#{i\in \text{qualifier}: \mu_i^{after\ next\ stage} < C_{cut}-10}
}{
#\text{qualifier}
}
]

意思是：一个已经晋级的队，在下一赛段结束后，掉到本地晋级切线以下太多的比例有多高。这个值如果还高，说明你的阶段保护不够。

第三，再加一个“国赛 0-3 中位掉分”监控。
我建议你把目标压在 15–35 分这个区间。
如果 0-3 普遍掉 60 分以上，说明你还在重复惩罚高阶段队伍。
如果几乎不掉分，说明保护过强。

八，如果你必须在旧代码上最小改动。

那就先别重写整套 shrinkage，先加“资格伪对局”。

做法是：
进入复活赛，给 1 个“战胜复活赛切线虚拟队”的伪胜场；
直通全国赛，给 2 个“战胜全国赛切线虚拟队”的伪胜场；
复活赛突围到全国赛，再给 1–1.5 个伪胜场。

这是简化版。它不如上面的阶段下界干净，但会立刻缓解“晋级反而掉分”的问题。

我对你的最终建议可以压成一句话：
赛前用多源加权先验，赛内只用真实对局更新，赛段结束后用“晋级认证下界 + 小样本 shrinkage”保护高阶段队伍。这样 0-3 会掉分，但不会被错误打成弱队。
