# 项目记忆

## 用户的一般要求

- 这个项目的前端不是普通数据后台，而是 RoboMaster 赛程模拟站，视觉必须有明确的赛事气质和机甲感。
- 用户对“泛用深蓝科技风”不满意，要求画面更贴近 RoboMaster 的红蓝对抗语义，红色必须足够明显，胜负态必须一眼可判。
- 配色不能脱离赛事语境。后续调色应优先参考：
  - `D:\code\douququ\微信图片_2026-04-14_221134_519.png`
  - [schedule.scutbot.cn](https://schedule.scutbot.cn/)
  - [robomaster.com/live](https://www.robomaster.com/live)
- 参考站只能学习结构、排布和信息表达，不做视觉克隆。
- 用户不接受“为了方便实现而牺牲赛程可读性”的设计。队伍名、比分、胜率、晋级/淘汰状态必须优先保证可读。
- 站内面向中文用户，前端展示不要暴露英文内部枚举。`Swiss`、`champion`、`national_qualified` 这类内部值都应翻译成中文。

## 已确认的产品结构

- 首页 `/` 是 RoboMaster 总控首页，不是简单表格页。
- 赛区页 `/regions/[region]` 使用单画布工作区。
- URL 兼容约束必须保留：
  - `seed`
  - `highlight`
  - `view`
- 后端接口语义不要改：
  - `GET /api/overview`
  - `GET /api/regions/:regionSlug/simulation?seed=...`

## 已确认的赛程展示原则

- 瑞士轮、资格赛、淘汰赛、结果回看是独立视图，不混成一个大杂烩。
- 资格赛和淘汰赛必须拆开看。
- 资格赛不能把全部晋级/淘汰结果堆到最后，应该像参考页一样“每一轮结束后就出现该轮分流结果”。
- 淘汰赛页只保留主淘汰链，不混入资格赛结果。
- 冠军战必须在季军战上方。
- 瑞士轮和结果回看里的长队名、长状态文本必须完整显示，不允许最后一行被裁掉。

## 已总结的实现经验

- 对这类赛程页面，静态画布比通用流程图库更合适。当前实现方向是自定义绝对定位卡片 + SVG 连接线，而不是 ReactFlow。
- 大改布局前应先用浏览器实际查看参考站，再决定画布分栏和连线方式。
- 这类页面仅靠代码阅读不够，必须用浏览器验收遮挡、层级、连线和缩放后的可读性。
- 当前环境下 `next dev` 偶发 `.next` chunk 损坏，出现 `Cannot find module './xxx.js'` 时通常不是业务代码问题，清理 `.next` 或直接用 `next start` 验证更稳定。

## 后续继续设计时的偏好

- 更强调红蓝对抗，不要把整站压成单一蓝绿色。
- 胜者、败者、晋级、淘汰必须有高对比状态设计。
- 画面可以有直播感和赛事气氛，但不能牺牲信息密度和结构清晰度。
- 如果要继续改主题，应整体改：全局变量、卡片层次、状态色、连接线、首页模块和赛程页控件一起调整，不要只局部换几个颜色值。
